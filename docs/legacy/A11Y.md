# FactoryMind — Accessibility (WCAG 2.1 AA)

**Version:** 1.0.0
**Standard target:** WCAG 2.1 AA (UE Dir. 2016/2102 transposed by D.Lgs. 106/2018 "Stanca law").
**Scope:** Public landing page (`landing-page/`), React dashboard (`frontend/`), documentation (`docs/`). Grafana dashboards inherit the Grafana a11y posture upstream.
**Reviewed:** 2026-04-18.

This document is the manual verification protocol that complements the
axe-core automated assertions in CI. Where automated tools can catch
~30-40% of WCAG issues, the remainder (focus-order, alt-text
appropriateness, keyboard traps) must be audited by a human. This
checklist is the canonical record of that audit.

## 1. Testing matrix

| Scenario | Tool | Frequency |
|----------|------|-----------|
| Automated axe-core smoke | `@axe-core/playwright` in CI | every PR |
| Lighthouse a11y audit | Playwright run + lhci | nightly |
| Manual keyboard-only walkthrough | Human + `docs/A11Y.md` checklist | per release |
| Screen-reader (NVDA / VoiceOver / Orca) smoke | Human + screen reader | per release |
| 200% zoom + 400% zoom layout | Browser dev-tools | per release |
| High-contrast mode | OS-level high-contrast | per release |
| Reduced-motion preference | `prefers-reduced-motion: reduce` emulation | per release |

## 2. Landing page checklist (`landing-page/index.html`, `landing-page/styles.css`)

### 2.1 Perceivable

- [ ] Every `<img>` has meaningful `alt` (or empty `alt=""` for decorative).
- [ ] Hero H1 is visible within the first 1.8 viewport heights (no dense overlays).
- [ ] Colour contrast: body text vs background >= 4.5:1, large headings >= 3.0:1.
- [ ] Steel-600 (#355269) vs white (#ffffff): 7.0:1. PASS. Safety-500
  (#facc15) vs steel-900 (#0f172a): 13.4:1. PASS.
- [ ] Do not rely on colour alone — alarms/CTAs use colour + icon + label.
- [ ] All videos (if any) provide captions.
- [ ] Decorative images (`role="presentation"` or `alt=""`).

### 2.2 Operable

- [ ] Tab order flows top-to-bottom, left-to-right without skipping.
- [ ] Skip-link present (`<a class="skip-link" href="#main">Vai al contenuto</a>`), visible on focus.
- [ ] All interactive elements focusable with visible focus ring
  (`:focus-visible` styles applied).
- [ ] No keyboard trap in any widget.
- [ ] `prefers-reduced-motion` respected: animations disabled via media
  query in `styles.css`.
- [ ] No auto-playing audio / video with sound.
- [ ] Forms can be submitted via keyboard Enter key.

### 2.3 Understandable

- [ ] `<html lang="it">` declared.
- [ ] `<meta charset="UTF-8">` declared.
- [ ] Form inputs have visible `<label>` (not placeholder-only).
- [ ] Errors flagged with `role="alert"` + human-readable text.
- [ ] Consistent nav across all pages.

### 2.4 Robust

- [ ] Valid HTML5 (`vnu` validator clean).
- [ ] ARIA roles used only when native semantics insufficient.
- [ ] `aria-live` regions for dynamic alerts (Mosquitto disconnect banner).

## 3. React dashboard checklist (`frontend/src/**/*.tsx`)

### 3.1 Structure & landmarks

- [ ] Single `<h1>` per page, hierarchical heading levels.
- [ ] `<nav>` for top nav; `<main>` wraps the route content.
- [ ] ARIA live region for new alert notifications (`role="status"`).

### 3.2 OEE Gauge (`components/OEEGauge.tsx`)

- [ ] Numeric value exposed via `aria-label` (the percentage read out loud).
- [ ] Colour threshold also communicated as text (e.g. "68% — World-class").
- [ ] SVG has `role="img"` + `<title>` fallback.

### 3.3 Alert Feed (`components/AlertFeed.tsx`)

- [ ] List semantics (`<ul>` / `<li>`).
- [ ] New entries prepended; use `aria-live="polite"` to avoid
  interrupting the user mid-read.
- [ ] Each entry has severity as text ("Critico", "Maggiore",
  "Avviso") in addition to colour.
- [ ] Acknowledge/Resolve buttons are `<button>` (not `<div>`), keyboard
  operable, with clear labels.

### 3.4 Charts (`components/DowntimeChart.tsx`, Recharts)

- [ ] Data table alternative view (`<table>` toggle) for screen readers.
- [ ] `aria-label` on the chart container naming the metric + period.
- [ ] Legend text uses high-contrast colours.
- [ ] Tooltip content keyboard-accessible (not hover-only).

### 3.5 Forms (`pages/DeviceConfig.tsx`)

- [ ] Every `<input>` has an explicit `<label for>`.
- [ ] Validation errors announced (`aria-describedby` pointing to the
  error span, `role="alert"` on submit failure).
- [ ] Required fields marked with `aria-required="true"` + visual "*".

### 3.6 Navigation (`App.tsx`)

- [ ] `NavLink` active state conveyed via `aria-current="page"`.
- [ ] Focus moves into `<main>` after route change (via
  `useEffect(() => focusRef.current?.focus(), [pathname])`).

## 4. Keyboard walkthrough

Protocol: with mouse unplugged, complete each flow using only keyboard.

1. Load `/` landing page.
2. Tab to skip-link; activate with Enter; focus should jump to `<main>`.
3. Tab through nav, sections, CTA buttons. No focus loss.
4. Land on `/login` page from dashboard app. Tab into email, enter,
   password, enter, submit.
5. Navigate to `/` (Dashboard) post-login. Use arrow-keys in any chart
   data table.
6. Navigate to `/alerts`; acknowledge the top alert with Enter.
7. Open `/devices`; edit a device via the form; save.
8. Return to `/`; verify that the live MQTT connection indicator
   announces changes to a screen reader.

Each step is logged in `docs/a11y-audit/<date>.md` alongside any failures.

## 5. Screen-reader smoke test

Protocol: with the screen reader running (NVDA on Windows / VoiceOver
on macOS / Orca on Linux), perform the keyboard walkthrough above.
Note any:

- Unlabelled form fields.
- Buttons read out as "button" (no accessible name).
- Charts read out as "image" with no meaningful description.
- Duplicate heading levels.
- Live regions announced too aggressively or not at all.

## 6. Zoom 200% / 400% layout

- [ ] Landing page reflows without horizontal scroll at 200%.
- [ ] Dashboard layout (Tailwind grid) reflows at 400%; chart panels
  stack vertically; nav collapses into a burger with `<details>`.

## 7. Reduced-motion + high-contrast

- [ ] CSS respects `@media (prefers-reduced-motion: reduce)` —
  `landing-page/styles.css` must contain this block.
- [ ] High-contrast mode (forced-colors) keeps focus rings visible.

## 8. Known gaps (to close)

1. Recharts tooltip is hover-triggered by default. Workaround: show data
   table equivalent via toggle button.
2. Grafana upstream contrast on some panel types < 4.5:1; overridden by
   `grafana/provisioning/dashboards/*` panel-specific theming where
   possible.
3. MQTT disconnect banner animation uses opacity pulse; the
   `prefers-reduced-motion` suppression is applied via CSS but not yet
   tested under Orca.

## 9. Automated CI assertions

- `@axe-core/playwright` asserts 0 a11y violations on `/` landing page
  and on the three dashboard routes (`/`, `/alerts`, `/devices`).
- Violations with severity `critical` or `serious` fail the build.
- Violations with severity `moderate` or `minor` post a comment; owner
  triages next sprint.

## 10. Review cadence

Manual a11y audit runs:
- Every release candidate (RC).
- Quarterly re-audit even without feature churn.
- Immediately on any user-reported a11y bug.

Audit records: `docs/a11y-audit/<YYYY-MM-DD>.md`. Each record contains
the checklist above with PASS / FAIL per row + free-form notes +
screenshots.
